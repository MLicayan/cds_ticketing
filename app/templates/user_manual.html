CDS Service Monitoring – User Manual

Overview

Web app for tracking tickets, service logs, PM/weekly schedules, instruments, and reports.

Key data: Tickets, Service Logs, Schedules, Instruments, Reports.

Roles & Permissions
Admin
Full access; manage users (admins/engineers/sales/clients), clients, instruments, apps, parts, service types; view all data.
Engineer / IT
Full access to tickets and service logs; can create/edit logs, schedules, exports, reports.
Sales
Create tickets and view modules; cannot change status/priority/assignee or create service logs.
Client
Create tickets for own client; view own tickets/logs; cannot edit status/assignee/priority; must approve closures with a signature.
Navigation
  • Dashboard: KPIs scoped by role/client.
  • Tickets: list, filters, Gantt, calendar, kanban, CSV export.
  • Service Logs: list, filters, Gantt, CSV/PDF.
  • Weekly / PM Schedules: plan work; link PMs to service logs.
  • Instruments: inventory and per-instrument logs.
  • Reports: top instruments/engineers (admin/engineer).
  • Admin: manage users, clients, instruments, parts, apps, service types.
Ticket Workflow
  • Create: Tickets → New; pick client + instrument/app, subject/description, optional photo; ticket no: T-000001 format.
  • View/Update: Detail shows status, priority, assignee, timeline, logs, attachments. Admin/Engineer can change status/priority/target date/assignee; Sales/Client read-only for these.
  • Comments: Clients/Sales add public comments; staff can mark Internal.
  • Close client-reported: Closing requires a ticket-level client signature (capture via modal before status = Closed).
  • Resolution: Service log with status_after=operational sets ticket to Resolved; non-operational keeps In-Process.
  • Views/Export: Gantt, Calendar, Kanban, Export CSV.
Service Log Workflow
  • Create (Admin/Engineer): Service Logs → New; select client, service-for (instrument/app), engineer, visit date, start/end time, status_after, confirmation photo, confirmed by/position; capture onsite signature for non-remote types; add parts; Save.
  • Links: Launch from ticket or PM to prefill.
  • Edit: Log detail → Edit Log (admin/assigned engineer) → update fields/times/monitoring, add attachments.
  • Exports: CSV from list; PDF per log.
  • Ticket sync: operational → Resolved; otherwise In-Process (if ticket not closed).
PM Schedules
  • Create: PM Schedule → New → client + instrument + date + description + assign engineer → Save.
  • Filter by client/instrument/date range; open detail → create linked service log when done.
Weekly Schedules
  • Create: Weekly Schedule → New → set week start/title → add tasks (client + instrument/app + subject/service type + engineer/priority) → Save.
  • Use tasks to drive work; tasks can spawn tickets where applicable.
Instruments
  • View list (filtered by client for staff; auto-scoped for clients).
  • Instrument detail: service history table, QR link, last PM/calibration.
  • Click a log row to open service log detail.
Reports
  • Admin/Engineer: view top instruments (visits/defects) and top engineers (log counts).
  • Client scoping applies automatically for client users.
Admin
  • Users: manage Admins, Engineers/IT, Sales, Clients (username, full name, password, role, optional client).
  • Clients: create/edit; map apps; add client users.
  • Instruments/Apps/Service Types/Parts: manage reference data.
Attachments & Signatures
  • Uploads stored per module (tickets, service logs, PM schedules).
  • Ticket closure (client-reported) requires a ticket-level signature captured on status change.
  • Service logs require onsite signatures for non-remote types.
Exports
  • Tickets: Export CSV (Tickets page).
  • Service Logs: Export CSV (logs list); PDF per log.
Admin Setup
  • Create users under Admin (Admins, Engineers/IT, Sales, Clients).
  • Configure upload folders: UPLOAD_FOLDER_TICKETS, UPLOAD_FOLDER_SERVICE_LOGS, UPLOAD_FOLDER_PM_SCHEDULES.
  • Legacy scrypt hashes on older Python are supported; new passwords use PBKDF2.
Troubleshooting
  • Cannot close client-reported ticket: capture a ticket signature in the closure modal.
  • Service log save blocked: check required fields (client, target, engineer, visit date, photo, signature for onsite).
  • Missing start/end time: edit the log; PDFs reflect these fields.
  • Permission errors: confirm role (Sales/Client are read-only for many actions).
